# Reading Notes 작업 기준

## 장 정리 형식

- 책의 장을 정리할 때는 기존 문서 흐름에 맞춰 `## n장 정리: 제목`, `### 메모`, `### 상세` 구조를 사용한다.
- `메모`에는 사용자가 준 핵심 문장을 짧은 불릿으로 정리한다.
- `상세`에는 메모를 다시 길게 풀기보다, 나중에 다시 읽었을 때 개념이 떠오를 정도로만 보충한다.

## 상세 작성 기준

- `상세`의 하위 주제는 `####` 제목을 쓰지 않고, `- **주제명**` 형식으로 나눈다.
- 하위 주제에 속한 설명 문단은 목록 아래에 들여쓰기해서, 어느 주제에 속하는지 문서 뷰어에서도 분명하게 보이게 한다.
- 하위 주제 하나는 보통 2~4문장 정도로 짧게 유지한다.
- 원문 포인트가 적으면 상세도 짧게 둔다. 예시를 많이 추가해서 분량을 늘리지 않는다.
- 같은 내용을 `메모`와 `상세`에서 반복할 때는, `상세`가 단순 반복이 되지 않도록 인지적 이유나 읽는 관점만 짧게 덧붙인다.

## 문체

- 설명은 학습자가 나중에 기억을 되살릴 수 있는 자연스러운 문장으로 쓴다.
- 어려운 개념은 용어를 먼저 말하고, 바로 다음 문장에서 뜻을 짧게 풀어준다.
- 사용자의 정리 문장에 오탈자가 있어도 의미를 살려 자연스럽게 다듬는다.
